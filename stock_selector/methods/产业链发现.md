# 方法论卡 · 产业链发现（发现层）

> 用途：给定主题/方向，拆产业链、锁稀缺环节、挖候选公司。融合产业链五步(Serenity 提炼)+分析师视角。
> 铁律：本卡只产出"环节 + 候选名"，所有名字最终交 `verify` 数据校验；纯判断步骤须列证伪反例。

## 五步（每步标注 [数据] / [判断]）
1. **系统变化** [判断]：主题里"什么变量变紧了"(算力/材料/认证/产能…)，不停留在热度。
2. **拆产业链层级** [判断+数据]：上中下游；可对照 `recall` 返回的概念板块名锚定真实环节。
3. **锁稀缺环节** [判断]：供应商少 / 认证周期长 / 扩产难 / 替代难。**必须列出判断依据**。
4. **挖候选公司** [数据兜底]：① Claude 点名该环节公司 ② 同时让 `recall` 拉相关概念板块成分兜底 → 合并。
   ⚠️ 概念板块召回在本环境为 best-effort，可能返回空(数据源受限)；此时以 Claude 点名为主，**点名的每只仍必过 `verify` 校验**。
5. **证伪检查** [判断]：每个候选列 ≥1 条可证伪反例(如"若 X 不成立则证伪")。

## 输出给工具桥
- 选定的概念板块名列表(从 `recall` 的可选板块里挑；取不到则跳过) → 传 `recall --boards`
- Claude 点名的代码列表 → 传 `recall --llm`
