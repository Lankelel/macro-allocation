# 方法论卡 · 环节排序 chain_segment_ranker（主题↔个股之间的"环节"一环）

> 用途：拆完产业链后，**先横评各环节、选出"强环节+好格局"，再进个股**。补上 theme_decider 与 stock_selector 之间缺的中间层。
> 铁律：**先环节、后个股**——绝不在差环节里挑"最好的个股"（如某环节净利-74%/PE200+，再好的标的也性价比差）。

## 为什么要这一层
个股分析若孤立做，容易"矮子里拔将军"。把个股先放进**对应主题的产业链**，看它在**哪个环节**、该环节**盈利性/竞争格局/估值/景气**如何，再带着判断回到个股 → 避免买在透支/拥挤/景气未兑现的环节。

## 四步
1. **拆环节**（接 `产业链发现.md` 第 2 步）：上游(元器件/材料) → 中游(核心硬件，A股主战场) → 下游(应用/需求源头)。每个环节点 2-4 只代表龙头（带代码）。
2. **iFinD 取环节数据**：对每个环节的代表龙头，调 **iFinD MCP** `get_stock_summary`（一次最多 5 只）取：
   - 估值：PE(TTM)、PEG、PB；盈利：**ROE(TTM)、毛利率(TTM)**；成长：营收/净利同比增速；行业分类 + 主营构成。
   - ⚠️ **务必用 TTM 口径**：摘要默认 MRQ(最新单季)，单季 ROE/利润会严重失真（如某温控股季 ROE 仅 0.25%）。聚合到环节层时取 TTM 列。
3. **四维评分选环节**（景气 × 盈利 × 格局 × 估值）：
   - **景气**：环节营收/净利增速 + 下游 capex 趋势（持续性，不是一次性）。
   - **盈利质量**：ROE(TTM) / 毛利率(TTM) / 趋势。
   - **竞争格局**：集中度 / 壁垒（认证周期、良率学习曲线）/ 全球地位 / 国产化率。
   - **估值性价比**：PEG、PE 分位；**警惕"概念热但业绩没兑现+估值透支"**（高PE+净利下滑=最差，直接否）。
   - 输出：选 1 个"最可挖掘环节"（强景气+好格局+估值未透支），并标注被否环节的原因（拥挤/透支/薄利）。
4. **可视化（可选但推荐）**：把环节谱写成 JSON，调生成器出**产业链全景图**：
   `py -3.13 -m uv run python -m stock_selector.chain_map --spec <谱.json> --out outputs/<主题>产业链全景图.html`
   - 谱结构：`bands[上中下游]`→`segments[{name,star,rows[{k,v}或{k,leaders[{t,code,cls}]}]}]` + `why`(选定理由) + `scorecard`(环节估值×盈利速览表)。
   - 卡片四要素：**主要产品 · 作用(一句) · 龙头公司(带代码) · 大订单/客户**；选定环节 `star:true` 高亮红框。
   - chip 的 `cls`：`hot`=本环节龙头、`gov`=海外/未上市(非A股可投)。

## 交回主流程
- 选定环节 → 该环节的代表龙头代码列表 → 作为 stock_selector `verify/score` 的候选池（仍每只过个股门+决策弹药卡）。
- 环节即天然的 `--segments` 标注来源（喂给 basket 做"同环节≤1只"去重）。

## 证伪
- 选定环节须能被数据证伪：若 ROE/增速/PEG 不支持"强环节"判断，或下游 capex 见顶，则重选。曾据此否掉"高景气液冷"叙事(iFinD 实测净利-74%/PE200+)。
